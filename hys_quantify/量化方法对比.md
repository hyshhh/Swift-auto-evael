# Qwen3.5 量化方法对比

## 问题背景

Qwen3.5 模型不支持 GPTQ/AWQ 量化，需要使用其他量化方法。

## 支持的量化方法

### 1. BitsAndBytes (bnb) - 推荐

**优点：**
- 最常用，兼容性最好
- 支持 4-bit 和 8-bit 量化
- 支持 NF4 和 FP4 量化类型
- 支持双重量化，进一步压缩

**缺点：**
- 推理速度不如 GPTQ/AWQ

**配置参数：**
```bash
--quant_method bnb
--quant_bits 4
--bnb_4bit_quant_type nf4  # 或 fp4
--bnb_4bit_use_double_quant true
--bnb_4bit_compute_dtype bfloat16
```

**脚本位置：**
`examples/train/qlora/qwen3.5-2b-bnb-sft.sh`

---

### 2. HQQ (Half-Quadratic Quantization)

**优点：**
- 无需校准数据
- 量化速度快
- 支持多种位宽 (1,2,3,4,8)

**缺点：**
- 精度可能略低于需要校准的方法

**配置参数：**
```bash
--quant_method hqq
--quant_bits 4
--hqq_axis 0
```

**脚本位置：**
`examples/train/qlora/qwen3.5-2b-hqq-sft.sh`

---

### 3. Quanto

**优点：**
- 支持多种位宽 (2,4,8,float8)
- 来自 Hugging Face 生态

**缺点：**
- 相对较新，社区支持较少

**配置参数：**
```bash
--quant_method quanto
--quant_bits 4  # 或 2, 8, float8
```

**脚本位置：**
`examples/train/qlora/qwen3.5-2b-quanto-sft.sh`

---

### 4. EETQ

**优点：**
- 8-bit 量化，精度损失小
- 推理速度快

**缺点：**
- 仅支持 8-bit

**配置参数：**
```bash
--quant_method eetq
--quant_bits 8
```

---

### 5. FP8

**优点：**
- 最新的量化格式
- 精度高
- 支持新一代 GPU (H100, Ada Lovelace)

**缺点：**
- 需要特定硬件支持

**配置参数：**
```bash
--quant_method fp8
```

---

## 位宽对比

| 位宽 | 模型大小压缩 | 显存节省 | 精度损失 |
|------|-------------|---------|---------|
| 8-bit | ~50% | ~50% | 极小 |
| 4-bit | ~75% | ~75% | 轻微 |
| 2-bit | ~87.5% | ~87.5% | 较大 |

## 推荐方案

### 场景 1：追求兼容性
**使用 BitsAndBytes 4-bit**
```bash
--quant_method bnb --quant_bits 4
```

### 场景 2：追求速度
**使用 EETQ 8-bit**
```bash
--quant_method eetq --quant_bits 8
```

### 场景 3：无校准数据
**使用 HQQ 4-bit**
```bash
--quant_method hqq --quant_bits 4
```

### 场景 4：有 H100/Ada GPU
**使用 FP8**
```bash
--quant_method fp8
```

## 安装依赖

```bash
# BitsAndBytes
pip install bitsandbytes>=0.43.0

# Quanto
pip install quanto

# HQQ (通常已包含在 transformers 中)
pip install transformers>=4.40.0

# EETQ
pip install eetq
```

## 使用步骤

1. 选择合适的量化方法
2. 修改对应的脚本文件
3. 运行训练：
```bash
bash examples/train/qlora/qwen3.5-2b-bnb-sft.sh
```

4. 推理时使用量化模型：
```bash
swift infer \
    --adapters output/Qwen3.5-2B-bnb-qlora-sft/vx-xxx/checkpoint-xxx \
    --stream true
```
